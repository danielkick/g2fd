Sanitization functions

The pattern to use is: 1. Alter the dataframe 1. Test the dataframe against expectations

The main tasks that need to be completed are: 1. Identify values that can’t be converted to the expected data type. The “find_unconvertable_” family of functions should be used. 1. find_unconvertable_datetimes

  1. For simple renaming (e.g. misspellings) or splitting non-tidy data into two rows (“entry1-entry2” -> “entry1”, “entry2”) use sanitize_col
  2. Move values that are ambigous but pertain to data imputation to “Imputation_Notes” using relocate_to_Imputation_Notes
  3. If new columns need to be added (e.g. mgmt.Ingredient for parsed components of Product (e.g. elements) ) this should be accomplished with safe_create_col.
  4. Any one off changes should be accomplised manually.
  5. Confirm columns match the expected types with check_df_dtype_expectations, and report mismatches.

These steps should be completed for each dataframe in turn to minimize the cognitive load of the reader.
